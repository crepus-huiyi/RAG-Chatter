# RAG-Chatter
基于RAG（检索增强生成）技术的服装领域轻量化智能问答系统，解决通用大模型问答幻觉问题，实现贴合垂直业务场景的精准、可控智能问答服务。

## 项目简介
本项目是面向服装领域的垂类RAG智能问答系统，聚焦服装洗涤养护、尺码推荐、颜色搭配三大核心业务场景，完整实现了**知识库动态更新、智能语义检索、多轮对话记忆、问答精准可控**的全链路能力，可快速适配企业自有知识库的轻量化部署与定制化需求。

## 项目背景
通用大模型的开放式问答存在明显的「幻觉」问题，无法精准贴合企业自有业务知识库内容，难以满足服装行业垂直场景的标准化、精准化客户咨询需求。本项目基于检索增强生成（RAG）技术，构建了一套可落地、可扩展的轻量化知识库问答系统，约束大模型仅基于企业知识库内容作答，大幅提升问答准确率与业务适配性。

## 核心功能
- 结构化知识库管理：支持服装领域业务知识批量入库、文本智能分块、MD5去重过滤、向量持久化存储
- 精准语义检索：基于向量相似度匹配，实现用户问题与知识库内容的精准召回，可配置检索阈值过滤无关信息
- 可控式RAG问答：通过提示词工程强约束大模型仅基于检索到的知识库内容作答，从根源上解决幻觉问题
- 多轮对话记忆：支持对话历史按session_id持久化存储，实现上下文连贯问答，页面刷新/重启后可恢复历史对话
- 可视化交互界面：基于Streamlit实现轻量化Web交互，支持知识库文件上传、问答对话一站式操作

## 快速开始
### 1. 环境准备
- Python 3.10 及以上版本
- 已开通阿里云DashScope API与通义千问API访问权限，获取对应API Key

### 2. 克隆项目与安装依赖
```bash
# 克隆仓库
git clone https://github.com/crepus-huiyi/RAG-Chatter.git
# 进入项目目录
cd RAG-Chatter
# 安装项目依赖
pip install -r requirements.txt
```

### 3. 环境配置
在项目根目录创建`.env`文件，填写如下核心配置：
```env
DASHSCOPE_API_KEY=你的阿里云DashScope API Key
QWEN_API_KEY=你的通义千问API Key
VECTOR_DB_PATH=./chroma_db
CHAT_HISTORY_PATH=./chat_history
```

### 4. 启动项目
```bash
# 启动Streamlit交互服务
streamlit run app_qa.py
```
启动完成后，访问浏览器输出的本地服务地址，即可进入智能问答交互页面，上传知识库文件并开启对话。

## 核心模块实现
### 1. 知识库管理模块（knowledge_base.py）
- 文本预处理：通过`RecursiveCharacterTextSplitter`按段落/句子自然分割文本，设置`chunk_size=1000`、`chunk_overlap=100`，保证拆分后文本的语义完整性
- 文本去重：基于MD5哈希值实现文本内容唯一性校验，为每一段待入库文本生成MD5哈希值，记录至`md5.text`文件，上传文件时先校验哈希值，已入库内容直接跳过，避免向量库冗余
- 向量入库：调用阿里云DashScopeEmbeddings完成文本向量化，将生成的向量与元数据存入Chroma本地向量库，完成知识库与嵌入模型的交互绑定

### 2. 向量检索与匹配模块（vector_stores.py）
- 封装向量库操作能力，统一管理Chroma向量库的增删改查，兼容LangChain Retriever接口标准
- 优化检索策略：设置检索参数`k=1`、`similarity_threshold=1`，精准过滤无关内容，为RAG问答链提供高匹配度的参考资料
- 实现用户问题实时向量化转换与相似度匹配，毫秒级完成检索响应，保证问答链路的低延迟

### 3. RAG问答链核心模块（rag.py）
- 串联「检索-提示词-大模型」核心流程，构建RAG全链路逻辑闭环
- 定制标准化提示词模板，拼接「系统指令+参考资料+对话历史+用户问题」，强约束大模型仅基于知识库内容回答，无匹配内容时明确告知用户，拒绝编造信息
- 通过`RunnableWithMessageHistory`关联对话历史存储，实现多轮上下文连贯问答，完整保留用户咨询轨迹

### 4. 对话历史与状态管理模块（file_history_store.py）
- 封装`FileChatMessageHistory`能力，将LangChain BaseMessage对象转为JSON格式，按session_id分文件持久化存储到本地
- 提供`add_messages`、`clear`等方法，灵活管理对话生命周期
- 基于Streamlit `st.session_state`存储核心服务实例与对话状态，避免页面刷新后重新初始化，保证交互连续性
- 支持通过session_id恢复历史对话，即使关闭页面后再次打开，也能保证上下文统一

### 5. 配置与交互模块
- `config_data.py`：统一管理项目全局配置，支持环境变量加载，集中管理API密钥、路径参数、模型配置等核心内容
- `app_file_uploader.py`：实现知识库文件的上传、解析、预处理全流程，支持TXT等格式文件批量上传
- `app_qa.py`：Streamlit Web交互主入口，整合文件上传、知识库管理、问答对话全功能可视化操作

## 问题与解决方案
**问题**：多次上传同一TXT文件时，向量库中会生成重复的文本向量，导致检索结果出现重复内容，影响问答体验与检索效率。
**解决方案**：
1. 对每一段待入库的文本内容计算MD5哈希值，建立已入库哈希值的本地记录文件`md5.text`
2. 每次上传文件并完成文本分块后，先逐段校验MD5哈希值
3. 若哈希值已存在于记录中，直接跳过该文本块的入库流程；若为新内容，完成入库后同步更新哈希值记录
4. 从根源上避免了向量库的内容冗余，保证检索结果的唯一性与准确性
